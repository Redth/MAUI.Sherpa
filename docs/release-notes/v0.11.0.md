## 🏔️ v0.11.0 — "Building the Summit Cache"

The rope team has moved from scattered gear bags to a proper locked cache on the route.
This release gives MAUI Sherpa a centralized secrets vault, local encrypted storage, and a
cleaner way to move credentials between cloud providers without losing track of what belongs
where.

### 🆕 New Features

- **Centralized Secrets Vault** — Secrets now flow through a managed vault architecture with
  metadata, normalized paths, and provider-aware operations, giving the expedition one reliable
  inventory instead of a pile of separate caches. (#172)
- **Local SQLCipher Vault** — Sherpa can keep secrets in an encrypted local vault backed by
  SQLCipher, with secure key storage and first-run onboarding for climbers who want a private
  base-camp stash before syncing to the cloud. (#172)
- **Expanded Provider Trail Map** — OnePassword, Infisical, Vaultwarden, Azure Key Vault, AWS
  Secrets Manager, Google Secret Manager, and Azure DevOps now plug into the same provider
  model, making cloud secret routes easier to compare and manage. (#172)

### ✨ Improvements

- **Secrets Page Overhaul** — The secrets UI gained richer listing, editing, publishing, and
  provider-selection flows, so credential management feels less like bushwhacking and more like
  following cairns up the trail. (#172)
- **Encrypted Settings Upgrade** — Settings storage now has stronger encrypted-service coverage
  and migration-friendly behavior, helping existing gear make the climb into the new vault setup.
  (#172)
- **Secret Path Normalization** — New path helpers and tests keep provider-specific naming quirks
  from sending secrets down the wrong switchback. (#172)

### 🔧 Infrastructure

- **Vault Test Coverage** — The new vault, provider factory, path handling, secure storage, and
  managed secrets services are covered by a broad set of unit tests, tightening the anchors before
  the next ascent. (#172)
- **Secrets Storage Documentation** — A new architecture guide documents the local vault,
  provider model, and cloud synchronization flow so future route-builders can clip in safely.
  (#172)

---

*The cache is locked, labeled, and ready for the next team to climb higher without leaving
credentials scattered in the snow.* 🔐
