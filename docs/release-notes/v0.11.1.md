## 🏔️ v0.11.1 — "Checking the Vault Latch"

The team is still camped beside the new summit cache, but this patch release
tightens the latch before anyone trusts it with their most important gear.
Local secrets should now be easier to reach, safer to recover, and less likely
to send climbers wandering toward an unavailable vault key.

### 🐛 Bug Fixes

- **Default Local Provider** — Sherpa now consistently offers and activates the
  built-in Local provider for new secrets setups, even before the Local Vault
  introduction has been answered. (#173)
- **Local Vault Fallbacks** — If Local Vault access is denied or unavailable,
  Sherpa keeps the Local provider visible but can fall back to an existing
  remote provider instead of leaving the expedition roped to a locked cache.
  (#173)

### ✨ Improvements

- **Vault Access Recovery** — The Secrets and Settings pages now surface a clear
  Local Vault access notice with a grant-access action, giving users a marked
  trail back to local secrets when OS secure storage needs approval. (#173)

### 🔧 Infrastructure

- **Provider Selection Coverage** — New tests cover the Local provider default,
  declined-introduction behavior, unavailable vault access, and non-destructive
  local provider deletion so this anchor stays tight. (#173)

---

*A quick gear check at camp keeps the cache useful, the trail clear, and the
next climb moving without spilled secrets in the snow.* 🔐
