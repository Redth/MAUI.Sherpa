## 🏔️ v0.10.0 — "Setting Fixed Lines"

The scouting party from v0.9 has turned the next ridge into a proper route. This release anchors
the App Inspector outside the main app, teaches Sherpa to talk to newer DevFlow and Ailoha agents,
and swaps more of the expedition's footing onto official Microsoft platform packages.

### 🆕 New Features

- **Standalone App Inspector CLI** — External hosts can now launch the inspector without starting
  the full Sherpa app. The new `maui-sherpa-inspector` server publishes token-protected loopback
  URLs, a stable `INSPECTOR_READY` handshake, heartbeat-based shutdown, docs, and lifecycle tests
  so other tools can clip into the rope safely. (#169)
- **NuGet-distributed `sherpa-inspector` tool** — The inspector is packable as
  `Sherpa.AppInspector.Cli`, installable as a global .NET tool, published by CI on `v*` tags, and
  attached to GitHub releases as a tidy summit cache for automation. (#170)
- **Ailoha and DevFlow v1 inspector support** — Sherpa now discovers both standard broker ports and
  speaks legacy `/api/*` plus modern `/api/v1/*` protocols, keeping old DevFlow agents on the rope
  while newer Ailoha-style agents join the climb. (#167)
- **Accessibility metadata in the visual tree** — The inspector properties panel now surfaces MAUI
  `SemanticProperties.Description`, `Hint`, and `HeadingLevel`, making accessibility checks part of
  the main route instead of a side trail. (#150)

### ✨ Improvements

- **Xcodes.app-compatible Xcode selection** — Settings and CLI flows now support rename, no-rename,
  and optional symlink selection modes, so Sherpa can match the way your local Xcode camp is already
  pitched while keeping the old rename behavior as the default. (#166)
- **Longer-lived Apple Developer sessions** — Saved Apple download sessions now preserve full cookie
  metadata, restore legacy sessions, refresh near-expiry cookies, and only clear credentials when
  Apple actually rejects them. Fewer surprise re-auth climbs, more time moving gear. (#171)
- **Unified MAUI DevFlow workflow** — Repository docs, skills, package references, and debug guidance
  now point at the `Microsoft.Maui.Cli` / `maui devflow` trailhead while preserving existing Sherpa
  inspector compatibility. (#164)

### 🐛 Bug Fixes

- **macOS DevFlow startup privacy crash** — The standalone macOS head now includes the Bluetooth
  privacy usage string required by newer DevFlow agent builds, avoiding a TCC faceplant at launch
  when CoreBluetooth-backed discovery is present. (#165)

### 🔧 Infrastructure

- **Official Microsoft macOS platform packages** — The macOS project moved from community
  `Platform.Maui.MacOS` packages to `Microsoft.Maui.Platforms.MacOS`, including namespace updates,
  MAUI package alignment, and CI build updates. (#165)
- **Official Microsoft Linux GTK packages** — The Linux GTK head moved from `redth/Maui.Gtk` package
  IDs to the `dotnet/maui-labs` package line, with shared-source cleanup and feed configuration for
  the current preview wave. (#161)
- **Inspector build and release pipeline** — App Inspector CLI tests now run in CI, NuGet packages
  are packed as release artifacts, and tagged builds can publish the tool directly to NuGet.org.
  The fixed lines are now part of the supply chain, not just the route map. (#170)

---

*The anchors are set, the fixed lines are taut, and the next stretch of ridge is ready for a
bigger rope team.* 🧗
