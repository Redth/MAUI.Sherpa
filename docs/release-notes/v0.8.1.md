## 🏔️ v0.8.1 — "Tightening the Guy Lines"

High camp is stocked, but before the next push you still walk the perimeter, tighten the lines,
and make sure nobody trips over the gear in the dark. `v0.8.1` is that kind of patch release:
more flexible Xcode setup, clearer native navigation, steadier Android Doctor repairs, and a
critical fix for Intel Macs carrying the universal build.

### 🆕 New Features

- **Side-by-side Xcode installs** — Sherpa now handles multiple versioned Xcodes under
  `/Applications`, keeps `/Applications/Xcode.app` pointed at the selected default, and keeps
  `xcode-select` marching in step. (#134)

### ✨ Improvements

- **Native titlebar navigation** — Windows titlebar Doctor and Settings buttons can now drive
  Blazor navigation directly, so the route to the right tool is shorter and better marked. (#137)
- **Clearer Apple sign-in guidance** — Xcode Management now raises a visible sign-in banner when
  Apple Developer auth is missing, with a one-click path back into the existing sign-in flow.
  (#146)

### 🐛 Bug Fixes

- **Doctor Android SDK repairs** — The Android SDK fix path now respects a fresh install target
  and persists the repaired SDK location, so Doctor does not send you back down to the wrong
  trail on restart. (#139)
- **Intel Mac universal launch crash** — Universal macOS builds now load the right
  architecture-specific managed assemblies, fixing the bad-format crash on Intel Macs. (#144)

---

*High camp is a little sturdier, the routes are a little clearer, and the rope team is better set for the next climb.* 🏕️
