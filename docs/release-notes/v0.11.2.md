## 🏔️ v0.11.2 — "Tuning the Camp Radio"

The team is still posted beside the summit cache, but this patch tightens the
signals and signposts before the next climb. Publish profiles now know how to
reuse the developer identities already packed in Settings, and Copilot is back
on the radio with the current SDK.

### 🆕 New Features

- **Publish Profile Identity Mapping** — Apple and Google developer identities
  from Settings can now be mapped directly into publish profiles, with picker
  modals that keep credential reuse on the marked trail. (#174)

### ✨ Improvements

- **Identity-Aware Publish Profiles** — Publish profiles now carry mapped
  identity fields through backup and secret resolution, and the list view shows
  Apple/Google identity counts so each route is easier to inspect before
  departure. (#174)

### 🐛 Bug Fixes

- **Copilot SDK Integration** — Sherpa now uses the current Copilot SDK APIs so
  responses deserialize correctly, sessions are started before sending, and the
  in-app assistant can answer from camp again. (#175)
- **macOS Apple Tooling Privacy** — The native macOS head now declares the
  Bluetooth usage string required by Apple development tooling, avoiding a TCC
  slip while devices are being observed. (#174)

### 🔧 Infrastructure

- **Copilot SDK Update** — Core and native macOS now reference
  `GitHub.Copilot.SDK` 1.0.2, keeping the integration clipped into the current
  CLI protocol. (#175)
- **Publish Profile Mapping Coverage** — New tests cover Apple and Google
  identity mapping through publish profile secret resolution, tightening this
  anchor before it carries real release gear. (#174)

---

*The cache is still secure, the route cards are clearer, and the camp radio is
finally crackling back with useful guidance.* 📻
